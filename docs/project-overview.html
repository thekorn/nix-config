nix-config

A declarative, multi-machine Nix configuration managing 7 hosts across macOS and Linux. Built on Nix Flakes, nix-darwin, NixOS, and home-manager — one repo controls system packages, user dotfiles, desktop environments, and remote deployments for every machine.

04

Key patterns

Architectural decisions that keep the config maintainable across 7 machines and 2 operating systems.

Two-layer config per host

Each host file configures both system-level (boot, networking, SSH, Homebrew) and user-level (dotfiles, shell, programs) in one place. System config flows through nix-darwin or NixOS modules; user config through home-manager.

hosts/darwin/*.nixhosts/linux/*.nix

Composition through profiles

Home-manager profiles (darwin.nix, linux-server.nix, linux-vm-desktop.nix) bundle common modules. Host files compose profiles with machine-specific overrides.

home/shared/profiles/

Self-contained program modules

Each program has its own module in home/shared/programs/ with custom Nix options (e.g. custom.ghostty.fontSize, custom.git.commitMessageTool). Modules are imported conditionally per profile.

~50 modulescustom options

Work vs. personal separation

User IDs (thekorn / d438477), Homebrew splits (private.nix / work.nix), and home-manager modules (private.nix vs work.nix) keep personal and work configs cleanly separated.

flake.nix usershomebrew splits

Vendored packages for stability

Four packages are vendored in pkgs/vendored/ to work around upstream issues: Apple Container, forked gptcommit, hunk (prebuilt), and tmux 3.4 (pinned for /bin/sh regression).

pkgs/vendored/overlay

Remote deployment via flake apps

The flake exposes shell-app scripts wrapping nixos-rebuild switch with SSH targeting. Deploy any Linux host from any machine: nix run .#deploy-thekorn-server.

flake outputsSSH

07

Notable details

Small decisions that add up to a cohesive system across all machines.

Caps Lock → Control on all macOS

All three Darwin hosts remap Caps Lock to Control via nix-darwin system preferences.

Ghostty auto-launches tmux

The Ghostty terminal module wraps every shell session in tmux, with SSH sessions getting a distinct red theme to differentiate from local.

Europe/Berlin everywhere

All Linux machines share en_US.UTF-8 with German regional settings (LC_ADDRESS, LC_MONETARY, etc.) and Europe/Berlin timezone.

nix.enable = false on two hosts

thekorn-studio and BFG-043556 manage Nix through the standalone installer rather than nix-darwin's own Nix management.

Wayland desktop in a VM

thekorn-vm-desktop runs a full Hyprland compositor with SDDM, Waybar, Wofi, and screenshot tools — software-rendered with no GPU.
